# LoRA Preflight

キャラLoRA学習の前準備（画像整形 → タグ付け → AI Toolkit投入前データ作成）を、
自分のPCの中だけで行うツールです。ブラウザで操作しますが、インターネット上には
何もアップロードされません。

最終更新: 2026-07-06

---

## 1. はじめて使う人向けセットアップ

### 必要なもの

- Windows PC
- **Python 3.11 または 3.12**（入っていなければ https://www.python.org/downloads/ から。
  インストール画面で **「Add python.exe to PATH」に必ずチェック**）
- 初回セットアップ時のインターネット接続（部品の自動ダウンロードに使います）

### 手順

1. 受け取った `lora_preflight_app` フォルダを、好きな場所に置く
   （日本語のフォルダ名・OneDrive内でも動きます）
2. フォルダ内の **`run_windows.bat` をダブルクリック**
3. 初回は「Creating local Python environment...」と出て数分待ちます
   （2回目からはすぐ起動します）
4. 自動でブラウザが開きます。開かなければ手動で `http://127.0.0.1:7869/` を開く
5. 終了するときは黒い窓を閉じる（またはCtrl+C）

### タグ付け機能を使う場合（画像整形だけなら不要）

タグ付けにはAIモデル（約1GB）が必要です。どちらかでOK:

- 配布フォルダに `models/wd-eva02-large-tagger-v3/` が入っていればそのまま使えます
- 無ければ、タグ付け画面の **「初回ダウンロード」ボタン**を押す（インターネット必要）

## 2. 画面と使い方

画面は3つ。右上のボタンで行き来できます。

### 画面1: 画像整形（トップページ `/`）

LoRA学習用の規定サイズ（1024x1024 など）へ画像を整えます。

1. **入力画像フォルダ**を「参照…」で選ぶ → **画像一覧表示**
2. 画像ごとに種類を選ぶ:
   - **通常画像**（顔アップ・上半身など）… そのまま。1枚→1枚に整形されます
   - **全身絵** … カードの **「全身絵として処理（4枚生成）」にチェック**。
     1枚から4枚（`名前_1`〜`_4`）が作られます:
     | ファイル | 内容 | 用途 |
     |---|---|---|
     | `_1` | 頭から上半身 | 顔・髪の学習 |
     | `_2` | 首から下（顔なし） | 服・体型の学習 |
     | `_3` | 足元 | 靴・裾の学習 |
     | `_4` | 全身 | 全体バランスの学習 |
3. 全身絵はサムネに**赤い線**が出ます。**線をドラッグして首（顎の下）に合わせる**と、
   そこから下が `_2` になります。サムネをクリックすると拡大表示になり、
   **拡大中も線をドラッグできる**ので正確に合わせられます
4. **余白 ⇔ 切り取りバランス**のスライダー:
   0に近いほど余白を足して画像全体を残す / 1に近いほど切り取りで合わせる。
   迷ったら0.5のまま。切り取りが大きくなりすぎる画像は自動で余白側になります
5. **出力フォルダ**を選んで（空欄なら `入力フォルダ名_prepared`）→ **画像整形を実行**
6. 整形結果のサムネが画像ごとに表示されます。**クリックで拡大**して確認。
   画面のサムネ＝実際の出力ファイルなので、見たままが保存されています

出力先の中身:

```text
出力フォルダ/
  images/          ← 整形済みPNG（ここを見る）
  manifest.json    ← どの画像から何をどう作ったかの記録（普段は見なくてOK）
```

### 画面2: タグ付け・出力作成（`/tagging`）

整形済み画像にEVA02でタグを付け、学習用の `.png` + `.txt` セットを作ります。

1. **画像フォルダ**に整形済みの `images` フォルダを指定 → **サムネ一覧表示**
2. **タグつけ**を押す（1枚ずつEVA02が解析します）
3. キャラトリガー・共通タグ・管理トリガー（ジャケットあり等）を設定し、
   サムネでチェックを付ける
4. **AI Toolkit投入前まで作成** → 出力フォルダの `dataset/` に
   `001.png`+`001.txt` 形式で出ます

※「AI Toolkitへ流し込んで開始」ボタンは未実装（今後の課題）です。

### 画面3: 辞書編集（`/dictionary`）

タグの語彙フィルター辞書（`config/tag_dictionary.json`）を編集します。
**このファイルは消さないでください。**

## 3. 覚えておくと便利なこと

- フォルダ欄・スライダー等の設定は**自動で記憶**されます。次に開いたときも残っています
- 「参照…」のフォルダ選択ウィンドウが見当たらないときは、**ブラウザの後ろに
  隠れている**ことがあります。タスクバーを確認してください
- 拡大表示は「画像の外側をクリック」か「Escキー」で閉じます
- アップスケーラー（Real-ESRGAN等）は既定でオフ。必要な人だけ画面4の設定で有効化
  （詳細は下の「アップスケーラー」）

## 4. うまくいかないとき

| 症状 | 対処 |
|---|---|
| `No Python at 'C:\ProgramData\anaconda3\...'` | 他のPCで作られた実行環境が残っています。今の `run_windows.bat` は自動で作り直すので、もう一度実行。直らなければ `.venv` フォルダを削除して再実行 |
| `Python was not found` | Python 3.11 を入れて（PATHにチェック）再実行 |
| `Package install failed` | Python 3.11/3.12 か確認 → `.venv` を削除して再実行。ネット接続も確認 |
| 画面の表示が崩れる・古いまま | ブラウザで **Ctrl+F5**（強制再読み込み） |
| タグ付けで「タグモデル未保存」 | タグ付け画面の「初回ダウンロード」を押す |
| ポート7869が使用中 | 黒い窓で `run_windows.bat` を閉じてから再実行。または `python app.py --port 7870` |

## 5. 他のメンバーへ配布するとき（ZIPの作り方）

1. `lora_preflight_app` フォルダをコピーして、コピー側から以下を**削除**:
   - `.venv/`（受け取った人のPCで自動再作成されます）
   - `.sessions/`、`__pycache__/`
   - 出力済みの `dataset/` や `〜_prepared` フォルダが紛れていれば
2. `config/default_settings.json` をメモ帳で開き、**自分のPCのパス**
   （`vocabularyRoots` のDドライブのパス等）が入っていたら消す（`[]` にする）
3. 残すもの（消さない）:
   - `app.py` / `preflight_core.py` / `run_windows.bat` / `requirements.txt` / `static/`
   - `config/tag_dictionary.json`（タグ辞書）
   - `models/`（入れると受け取った人がダウンロード不要。ただしZIPが約1GB増える。
     外してもOK＝各自「初回ダウンロード」ボタンで取得できます）
   - `wheelhouse/`（あればオフライン導入用。無くてもオンラインで入ります）
4. フォルダごとZIP圧縮して渡す
5. 受け取った人はこのREADMEの「1. はじめて使う人向けセットアップ」から

### 更新版（差分ZIP）を受け取ったとき

1. 中身を `lora_preflight_app` フォルダへ**そのまま上書きコピー**
   （`config/` と `models/` は更新ZIPに含めない運用なので、辞書や設定は消えません）
2. `run_windows.bat` を再起動 → ブラウザで **Ctrl+F5**

## 6. アップスケーラー（任意・上級者向け）

既定では使いません。必要な場合のみ画像整形画面の「4. アップスケーラー」で選択:

- **SD WebUI / Forge API**: WebUI/Forgeを `--api` 付きで起動しておく。
  URL `http://127.0.0.1:7860`、Upscaler名 `R-ESRGAN 4x+ Anime6B`（倍率1倍で品質補正）
- **Real-ESRGAN単体exe**: `realesrgan-ncnn-vulkan.exe` のパスを指定
- **手書きコマンド**: `{input}` `{output}` が自動置換されます

失敗した場合はアップスケーラーなしの整形画像がそのまま出力され、警告が表示されます。

## 7. 開発者向けメモ

- 整形ロジックの本体は `preflight_core.py`（UI非依存・Pillowのみ）。
  設計書は開発リポジトリの `docs/lora-preflight/design-image-processing.md`
- テストは開発リポジトリの `tests/preflight_core_test.py`（ロジック）、
  `preflight_app_e2e.py`（HTTP）、`preflight_ui_e2e.py`（ブラウザ表示）
- 画面サムネと出力ファイルは同一関数 `apply_plan()` の産物（WYSIWYG保証）。
  プレビュー専用の別処理を作らないこと
