## 0. プロジェクト要旨
商業アニメ「**尚善（しょうぜん, DANGUN PICTURES）**」第7話の **背景原図（genzu = 背景のラフ/レイアウト）** を、
**クリーンな白黒線画の叩き台**に整え、美術監督のレビュー素材にするツール。

- 生成: **GPT Image 2**（Higgsfield 経由。ローカルは公式 `higgsfield` CLI）。
- 美術ボードは **時代背景・構成物の参照**に使う（**色は参照しない**）。出力は白黒線画。
- 最終物: 生成結果を元PSDに「**AI原図修正**」レイヤーとして差し込み。リテイクは `_02`,`_03`。
- 規模: 1話 約300カット。いまは **GKV担当23本**でローカル一気通貫を検証中。

## 2. パイプライン現状（ここは“完成済み”。品質以外は触らなくてよい）
1. **prep**: PSD → 背景作画レイヤーだけ合成(`psd_export.export_background_layer`) → ヘッダー帯除去(`frame.strip_header`) → **GPT出力寸ぴったりの入力**(`image_aspect.build_input_image`)。
2. **生成**: `higgsfield generate create gpt_image_2 --image <input> --aspect_ratio <ar> --resolution 2k --quality high --wait --json`。
3. **finish**: 生成結果を原図画角へ戻す(`image_aspect.restore_output_image`)→元座標へ(`frame.paste_into_region`)→「AI原図修正」差し込み(`psd_export.insert_result_layer`)→台帳(`ledger`)。
- バッチ実行: `python -m genzu_fix.batch --genzu-dir <原図> --out <出力> --assignee GKV`（`docs/local-cli-runbook.md`）。
- **レジストずれは解決済み**: 入力=出力グリッドにしたので幾何的にズレない（`design-notes.md §20.6`）。残るズレは生成の描き直し由来。
- **レイヤー選択**: トップレベルで `BG*` → 無ければ `LO*` → 無ければ `背景`。`美監補足/SL指示/セル参考/Camera/参考/[type]/BOOK◯` は除外（`--include-book` でBOOKも含められる）。GKV23本は BG*/LO* で全数カバー。

## 5. 次フェーズの目標（このセッションでやること）
**カットごとに、プロンプトへ入れる「場面情報」を構造化して用意する**仕組みを設計する。

### 5.1 1カットに欲しい項目（仮説）
- **場所** (place): 例「復活の儀の部屋」「森の中」「道観の寝室」… ← 香盤表/コンテ/フォルダ名。
- **時間** (time): 朝/昼/夕/夜 ← 香盤表/ボード名。
- **天気** (weather): 晴/雨/霧 ← 香盤表/ボード名。
- **場面/状況** (what's happening): コンテのその枠の説明。背景に何が要るかの根拠。
- **除去対象** (remove): キャラ＋その持ち物/装備（QCルール: 人物に属するものは消す、環境＝家具/什器は残す。例: ベッドの枕は残す、人物が持つ本は消す）。`qc.py` 参照。
- **構成物** (structures): その場所に在るべき物（柱/扉/寝台/燭台/窓…）。BOOK◯やコンテから。
- **時代/様式** (era/style): 美術ボード＋美術設定から（中華風時代劇）。色は不参照。

### 5.2 情報源の対応づけ（どこから取るか）
| 項目 | 一次情報源 | 補助 |
|---|---|---|
| 場所 | 香盤表 / フォルダ名 | コンテ、`04.美術設定` |
| 時間・天気 | 香盤表 / ボード名(`naming.parse_board`) | コンテ |
| 場面・除去対象 | **絵コンテ**（カット枠の絵と注記） | シナリオ |
| 構成物 | 美術ボード / 美術設定 / BOOK層名 | コンテ |
| 時代・様式 | 美術ボード / `04.美術設定` | クライアント設定資料 |

> `naming.py` に `parse_board`（シーン/場所/時間/天気/改訂を抽出）と `build_board_index`/`match_board` が既にある。
> `runs/cut_board_map_ep7.csv` で **カット→ボードは確定済み**なので、ボード名から場所/時間/天気は機械抽出できる。
> 不足は「**カット→場面/除去対象**」で、ここが**絵コンテ依存**。＝このフェーズの主戦場。

### 5.3 想定アウトプット
- カット別の構造化情報 CSV/JSON（例 `runs/cut_scene_info_ep7.csv`: `cut, place, time, weather, situation, remove, structures, era, source`）。
- それを `batch.build_prompt` / `--prompts-dir <cut>.txt` に流し込み、カット個別プロンプトを自動生成。
  - 現状の `build_prompt`（`src/genzu_fix/batch.py`）は「共通文＋ボード名/シーンのヒント」だけ。ここを 5.1 の項目で強化する。
- 香盤表(xlsx)パーサ、コンテ(PDF)→ページ画像→Vision抽出 の小ツールが要るはず。

## 9. レビュー知見・要件（GKV結果フィードバック 2026-06-24）
黒江さんがGKV生成結果を見ての指摘。次フェーズの要件として最重要。

### 9.1 全体評価＝生成品質 0点（現状の核心的欠陥）
- 実体は「線をなぞって整えただけ」。①そのカットが何か（場面/残す物/消す物/構成物）を**入力に持っていない**、
  ②**美術ボードを実際に参照していない**（v1はボード名を文章ヒント化しただけ＝モデルは中身を知らない）。
- → 品質の鍵は **(A) カット理解の付与** と **(B) ボードを実画像/具体情報として参照**。この2つが本フェーズの主目的。

### 9.2 フレーム切り取り禁止（再・最優先のパイプライン修正）
- `frame.strip_header` が頭をほぼ一律14%切るため、**画面全体を使うカット/上方向PANの over-scan**を削って台無しにする。
- 要件: **原図の描画範囲は切らない**。ヘッダーは「在るときだけ・文字帯の最小限」、不確実なら切らない(0px)。流背は全幅保持。
- これはプロンプトでなくパイプライン修正。`strip_header`/`header_bottom` を保守的化 or 既定オフ＋明示指定へ。

### 9.3 レイヤー選択の誤爆 → 手動オーバーライドが必要
- 原図のレイヤー名が**スキャン時の名前のまま**で雑なケースがあり、名前ベース選択が誤爆する:
  - **025**: 正解 `LO_SL指示_B1.jpg`（だが "指示" を含むため `_EXCLUDE_HINTS` で誤除外）。
  - **052**: 正解 `A2_B2_C1.jpg`（セル名のまま）。`BG_le` を誤選択。
- 対策: visible/body を目視 → **カット別にレイヤー名を指定できる手動オーバーライド**（CSV等）。`export_background_layer` に
  「このカットはこのレイヤー」を渡せる口を足す。名前部分一致の除外は弱いと割り切る。
- **030**: 説明用の赤字図解が背景レイヤーに**直書き**＝自動分離不可。手動消し（消しゴム）前提のカットとして扱う。

### 9.4 特殊背景の分岐（type を持たせる）
- **流背（流し背景）**: フレームより広い連続背景（PAN/TU用）。「BG①流背(2フレーム分)」等。全描画域必須・超ワイド比。
  例 031, 046。検出: シーン/フォルダ名・[type]注記の"流背/流し"・極端アスペクト比。扱い: 全幅保持／分割 or 強letterbox。
- **イメージ背景（心象/印象）**: 実在場所でなく抽象・感情の背景。リアル建築を当てない。フォルダ「イメージc290-292」・回想。
- → 入力前に **type(通常/流背/イメージ)** を判定して処理とプロンプトを分岐する。

### 9.5 参考画像・手書き指示の活用（リテイク含む）
- **035**: 原図がラフでLOがパースを直している＝**LO基準**。さらに「木組み方参考」レイヤーを**参照画像**として使いたい。
  → 原図内の「◯◯参考」レイヤーを抽出し、生成/リテイク時の**参照入力**に回す設計。
- **036**: 「BG全カット寄り」等の**原図内手書き指示**を読み、かつ 035 を正しく参照する必要。
- **038/039**: キャラアップで背景情報が薄く、「ここは柱」「光源」等の**手書き指示**を汲む必要。
- → 原図内の手書き指示は **Vision/OCR** で読む。情報の薄いカット(032,042)は**コンテ必須**。

### 9.6 まとめ：本フェーズの作業優先度
- **A. パイプライン修正（即）**: フレーム切り取り廃止（9.2）、レイヤー手動オーバーライド（9.3）。
- **B. カット理解（中核）**: コンテVision＋香盤表 → place/time/weather/situation/remove/structures/**type(通常/流背/イメージ)**。
- **C. ボード参照強化**: 美術ボードを**2枚目入力（画像）**として渡す（色は不参照を維持）。
- **D. 参考活用**: 「◯◯参考」レイヤー・手書き指示・LO基準カットのリテイク経路。

### 9.7 2枚入力テスト（cut205, 切らない＋ボード2枚目参照）の知見（2026-06-25）
切らない方式＋美術ボード2枚目参照を MCP で実機テストした結果と、黒江さんの指摘:
- **成果**: ヘッダーを切らずにプロンプト除去が成立。ボード参照で森の構成が描かれ、前回の「線なぞり0点」から前進。出力2336×1744=入力グリッド一致（レジストOK）。
- **誤解の訂正**: cut205 の原図がスカスカなのは情報不足ではなく、**キャラがアップで乗るため背景を意図的に薄くしている**。`セル参考` フォルダ（PSD内・非表示）を見るとキャラ位置が分かる。→ こういうカットは「ボードで埋める」より**原図の構造尊重**＋**キャラ域は描き込み過剰にしない**。`セル参考` から**キャラ占有領域**を取得して、その範囲の捏造を抑える設計が要る。
- **問題1: ボードが構造を上書き**。原図右の**太い幹**が、ボード（細い木が林立）に引かれて**分割され背景化**。→ プロンプトで「**IMAGE1=構造の正。太い幹は1本のまま、木を増やすな。IMAGE2は様式/空気感のみ**」と縛る必要。構造の権威は常に原図。
- **問題2: 立体感・奥行きが乏しい**。均一線で、線の強弱・前後の重なり・空気遠近（奥ほど細く淡く）・幹の円筒感が無く平面的。→ プロンプトに「重なり/線の強弱(近=太濃,遠=細淡)/空気遠近/幹は輪郭で円筒に」を明示。ただしGPT Image 2の線画は立体表現に限界がある可能性、要見極め。
- 含意: **A/B方針（原図忠実 vs ボード起こし）はカット種別で切替**。キャラアップ/原図に意図がある回は忠実寄り、背景主役で原図が薄い回はボード起こし寄り。`セル参考`・コンテ・香盤表で種別判定。
