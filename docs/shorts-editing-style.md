## 結論: 編集の型（食レポショート）

**連続区間の切り出しではない。** 1〜4秒のマイクロカットを以下の順で組み直したモンタージュ:

```
① フック (0〜2秒)   … いきなり「食べている最中」or「料理ドン」から始まる。
                      顔+料理が同一フレーム。準備・移動・経緯は一切入れない。
                      例: 頬張って笑う / 山盛りを持って「このビジュやばい」
② 食べ物の官能ショット連打 … 接写(ECU)で1〜3秒ずつ。1ショット=1セールスポイント。
                      タレが糸引く / 黄身が割れる / 湯気 / 脂の照り / チーズ伸び / 麺リフト
③ リアクション挿入   … 食べ物ショットの合間に1〜2秒で顔を返す(交互リズム)。
                      目キラキラ / 頬張る / 至福の目閉じ / 嬉し泣き
④ 実食の音          … 咀嚼音・すすり音を前面に(ASMR的)。BGMより効果音が主役。
⑤ オチ (最後3〜5秒) … 空の丼 / 満腹で腹をさする(ギャグ) / カメラ目線ひと言 /
                      チャンネル登録して？ で即終了。余韻を引かない。
```

### 数値の目安

- 尺: **16〜34秒**（食レポ。チル系のみ48秒）
- カット長: **1〜3秒**が主体。長回しは「ゆっくりズームの飯テロ接写」(5〜6秒)のみ許容
- ショット構成比: 食べ物接写 ≈ 40〜50% / 顔リアクション ≈ 35% / その他(状況・オチ) ≈ 15〜25%
- セリフ: **ゼロ〜1フレーズ**。本編の長い独白・愚痴パートは全カット。
  入れるなら「カロリーってなんで美味しいんだろう」級のパンチライン1本のみ
- 冒頭は**本編の途中(クライマックス)から**。文脈説明なし

### 型のバリエーション

1. **王道食レポ型** (#1, #5, #6): フック→飯テロ接写⇔リアクション交互→オチ/CTA
2. **ASMR型** (#2): セリフ無し。すすり音+咀嚼音で押し切る。カット割りだけで飽きさせない
3. **物語圧縮型** (#3): 起承転結を1秒/カットで23カット。買って帰る→開封POV→実食→満腹ギャグ
4. **チル型** (#4): 食事なし版。環境音+ピアノ、感情の起伏(猫→星空→夜景)で48秒

## 横型本編から作る際の含意

1. **contiguous(連続)切り出しでは型にならない。** 本編の食事クライマックス帯から
   マイクロカットを8〜15個拾い、上記の順に**並べ替えて**結合する。
   （実装: `make_shorts.py` の `cuts` 配列 = モンタージュモード）
2. **縦型化は crop 一択。** 参照ショートは全てフルフレーム9:16(ぼかし帯なし)。
   このチャンネルのAI動画は被写体中央配置が多く、`focus_x` 微調整で成立する。
   飯テロ接写カットは特に crop 耐性が高い（料理が画面中央）。
3. **音声設計**: 本編の独白ナレーションはモンタージュで切れ切れになるため、
   セリフが乗っているカットは「フレーズが完結する範囲」で拾うか、捨てて効果音カットを優先。
4. **Personal Clipper への評価**: Clipper は発話ベースの連続区間切り出し+字幕が得意
   ＝この型(マイクロカット・モンタージュ・字幕なし)とはズレる可能性が高い。
   出力は「素材候補の提示」として使い、本命は自前モンタージュで作る。
