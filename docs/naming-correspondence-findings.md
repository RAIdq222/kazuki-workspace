# 原図↔美術ボード 対応関係の検証メモ（2026-06-22 / shz ep07）

## 0. 前提（重要な制約）
- Drive コネクタは **10MB超のファイルを取得できない**。ep07 原図PSDは最小16MB〜最大460MB、
  絵コンテPDFは92MB。→ **実ファイルの中身は web セッションに引き込めない**。
  実処理はユーザー環境（Mac/サーバー、Higgsfield CLI）で回す前提（設計メモ §8）と整合。
- したがって本メモは「ファイル名と命名規則から検証できる構造」を対象とする。

## 1. テキスト補足指示の扱い（原図側）
PSD の中身は直接見られないが、扱い方は **テキストが「どこにあるか」で二択**:
- **別レイヤーにある場合**（テキストレイヤー/指示レイヤー）→ PSD→PNG 書き出し時に
  `kind=='type'` や名前規則で**除外**して入力PNGを作る（`psd_export.export_with_overrides`）。
- **絵に焼き込まれている場合**→ 生成プロンプトで**無視/除去を指示**（南康郡で実証済み）。
- 判定法: 実PSDで `list_layers` を流せば、テキスト/指示レイヤーの有無と名前が分かる。
  既存の検証PSD(IK_MV)では `用紙/100Fr赤(枠)/tap/【演出】【監督】【作監】【総作監】(チェック)/線画/ラフ`
  のように **絵と指示・チェックが別レイヤー**だった。ep07 も同様なら自動除外しやすい。
- 次アクション: 代表2〜3枚で `list_layers` の出力をもらう → 除外ルール（名前/種別）を確定。

## 2. カット↔ボードの対応（名寄せ）— 検証結果

`src/genzu_fix/naming.py` で実ファイル名をパースして確認。

### ボード名は機械可読に構造化できる（◎）
`SZ#2#3_南康群_街並_清書` → tags=`[SZ#2#3]`, place=`南康群_街並`
`#6_山の中道教の寺院_R1_夕方 （雨）` → tags=`[#6]`, time=`夕方`, weather=`雨`, rev=`R1`, place=`山の中道教の寺院`
→ **シーン番号・場所・時間・天気・改訂**がほぼ取り出せる。

### 原図名はカット番号で識別（◎）
`shz_07_091_101_116_genzu` → ep=07, cuts=`[091,101,116]`（複数束）
`shz_07_239B_genzu` → cuts=`[239B]`（枝番）

### だが両者は直接マッチしない（＝ここが「大変」の正体）
- ボードのキーは **シーン番号**（SZ#1 / SZ#2#3 / #6 / #6#7 / b01_xx）。
- 原図のキーは **カット番号**（268, 091…）。
- **共通キーが無い**ので、ファイル名同士では結べない。
  → **「カット → シーン/場所/時間/天気」の索引が必須**（出どころ＝絵コンテ／香盤表）。

### さらに、シーンが分かっても1枚に決まらない
1シーンに対しボードは **場所 × 時間 × 天気 × 改訂** で多数ある。
例: `#6` だけで 花氏邸(全景/入口/広間/祭壇)・復活の儀の部屋(昼/夜/外廊下)・寺院(朝/昼/夕/夜/雨)・山中の池…
→ 最終的に1枚へ絞るには **場所＋時間＋天気** が要る。これは香盤表の列そのもの（設計メモ §4）。
→ つまり **香盤表は二重に必要**: ①カット→シーンの紐づけ ②時間/天気でボードを一意化。

### 命名の揺れ（要・名寄せ正規化）
- 南康**群** / 南康**郡**（表記ゆれ）
- 攫われた / 廃われた古屋敷（表記ゆれ）
- `SZ#6` / `#6`（タグ形式の不統一）
- タグ無しファイル（`森明けげ方` `寺院内 台所 食堂…`）

## 3. 結論（リフレーミング）
- **ボード側は自動で構造化できる**。難しさの本体は「カット → シーン/場所/時間/天気」の1索引だけ。
- その索引の出どころは **絵コンテ／香盤表**。＝ 美術ベテランへのヒアリング C15
  「このカットがどのシーンか、どう判別しているか」の答えがそのまま設計図になる。
- 当面の運用案: 索引が無い間は **半手動**（カット→場所を人がCSVで埋める）→ ボード側は
  パーサで候補提示。索引が揃えば自動結合に移行。
