# 原図修正プロンプト設計：ロードマップと現在地

> 最終目的＝**原図(背景レイアウト)を適切に修正するための GPT-image2 生成プロンプトを設計できるようになる**こと。
> その手前に2段階ある。次セッションは「今どこまで出来ていて、次に何をやるか」をこの1枚で掴む。
> （可変な状況は `docs/sessions.md`、コンテOCRの仕組みは `docs/conte-ocr-overview.md`。）

## 全体の流れ

```
[Stage1] 3観点で原図を解釈 ─統合→ 「どう解釈し・どう修正すべきか」= situation/remove
   観点1 原図を見たまま評価
   観点2 絵コンテから演出意図で場面解釈     ← コンテOCRはこの“入力データ作り”
   観点3 設定/シナリオから文化・時代・場面考証
                    │
                    ▼
[Stage2] 定型作業(枠線除去/キャラ除去/時間→線処理)と結合 → GPT-image2 用プロンプト
                    │（参照画像 IMAGE2 を1枚積む：原則そのカットの美術ボード）
                    ▼
                 生成(白黒線画ドラフト) → PSDへ「AI原図修正」
```

## 現在地（You are here）

- いま深掘りしている**コンテOCRは「観点2の入力データ作り」**で、詰めの最終段（縦棒の番号判定）。残り数%。
- **地図で最大の未検証ブロックは Stage1 の 1-4（3観点を実カットで通す）と Stage2-C（situation/remove 充足）。**
  ここが「原図をどう修正すべきか」という目的の中核なのに、まだ一度も実機で回っていない。

## クリティカルパス（目的に最短）

1. コンテ：縦棒の番号判定を着地 → review2 で1回通し確定（観点2の素材を固める）
2. **Stage1 を1カットで end-to-end 実行**（例 cut274 道観・食堂、原図あり）。3観点→統合→situation/remove とアラートが実際に出るか初検証（**心臓部**）
3. 統合結果を Stage2 に流す配線(1-5) → `prompt show --cut N` で完成プロンプト確認
4. 1〜2カット実生成し GLOBAL 文面（線質/除去の効き）を実測確定（2-A 保留解除）
5. 観点3の精度上げ（2-B：森・花家・星空の grounding 追加）は並行で後追い

## 保留タスク（レアケース・全体精度を優先）

- **観点1へ「原図の補足情報」を渡す** … 原図には2つの顔がある：
  - 生成の入力（Stage2 IMAGE1）＝ BG→LO→背景の**クリーン背景のみ**（セル参考/指示/Camera は除外）
  - 解釈の材料（Stage1 観点1）＝ 本当は**除外したレイヤー**（セル参考＝芝居/立ち位置、指示メモ、原図内のシーン参考枠）にヒントがある
  - やるなら `psd_export` のレイヤー選択で「生成用」「解釈用(除外分含む)」の2系統を出し、観点1にだけ補足ビューを渡す（生成には混ぜない）。
  - **保留理由**: レアケース。原図内の手書き参考の読み取り精度も難しい。まず全体精度の底上げを優先する。
