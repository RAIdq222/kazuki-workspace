# 画像加工で追加したいこと

作成日: 2026-07-02

この文書は、今後このアプリに追加したい画像加工仕様を、人間とChatGPT/Codexの両方が読みやすいように整理したものです。  
実装者は、まずこの仕様を読み、既存の画像整形処理を壊さないように段階的に実装してください。

## 前提

現在のアプリには、LoRA学習用の規定サイズ候補があります。

現時点の候補:

```text
1024x1024
1152x896
1216x832
1344x768
1536x640
```

縦横入替を許可する場合、以下のような反転サイズも候補になります。

```text
896x1152
832x1216
768x1344
640x1536
```

ここで書く `1024` や `1536x640` は固定値ではなく、原則としてアプリ側で設定された規定サイズに従うものとします。

## 大きな方針

画像は大きく2種類に分けて処理したいです。

1. 上半身、顔アップ、バストアップなどの通常画像
2. 全身絵

通常画像と全身絵では処理方法を変えます。

## 1. 通常画像の処理

対象:

- 顔アップ
- 上半身
- バストアップ
- 全身ではない画像

処理方針:

1. 入力画像の縦横比を確認する。
2. 規定サイズ候補の中から、画像に一番近い比率を選ぶ。
3. 余白を上下左右に増やし、画像に一番近い比率へ合わせる。これを候補1とする。
4. できるだけ少ない切り取りで、その比率へ合わせる。これを候補2とする
5. 候補2の切り取り面積が候補1のｘ倍以上であるならば、候補1を採用する。

## 1’. スライダーの用意
候補1と候補2の選択が可能なように、ユーザーに、このxの数値を決定させる。
標準を0.5として、0~1間を移動できるようにする。
ユーザーが迷わないような説明文を追加する。（0に近づくほど余白を用意する側になる）


### 縦長画像の場合

縦長画像では、基本的に横を少し削って規定比率へ合わせる。

目的:

- 頭や顔を大きく壊さない。
- 上下を削りすぎない。
- 画像の見た目に近い比率を選ぶ。

### 横長画像の場合

横長画像では、基本的に上下を少し削って規定比率へ合わせる。

目的:

- 左右を残す。
- 上下を必要最小限だけ削る。基本的には上側を削り、絵のない白を優先して削る
- 画像の見た目に近い比率を選ぶ。

### 削りすぎる場合

一番近い比率を選んだ結果、切り取り量が大きすぎる場合は、無理にその比率へ合わせない。

代わりに、もう一段ゆるい比率、つまり切り取り量が少なくなる候補を選ぶ。

考え方:

- 縦長画像で削りすぎる場合は、横方向に余裕が出る比率を選ぶ。
- 横長画像で削りすぎる場合は、縦方向に余裕が出る比率を選ぶ。
- 最終的には、切り取り量が少なく、かつ規定サイズに収まるものを選ぶ。

重要:

- 削りすぎるくらいなら、より余裕のある規定サイズ候補へ逃がす。
- 画面のサムネと実際の出力画像がズレないようにする。

## 2. 全身絵の処理

対象:

- 頭から足先まで写っている画像
- 立ち絵
- キャラクター全身の資料画像

全身絵は、通常画像と同じように1枚へ押し込むだけでは不十分です。  
全身絵からは複数枚の学習画像を作ります。

## 全身絵から作る画像

全身絵1枚から、以下の4枚を作りたいです。

1. 頭から上半身の正方形画像
2. 首から下の正方形画像
3. 足元側の正方形画像
4. 全身絵をできるだけ残した画像

つまり、全身絵1枚から4枚出力します。

## 全身絵の前処理

全身絵の場合、まず高さを基準値に合わせます。

基準:

```text
高さ 2200
```

方針:

- 全身絵はまず高さを2200に合わせる。
- 横幅はこの時点では強く考慮しない。
- 高さ2200へ拡大または縮小した後、各切り出しを行う。

この `2200` は将来的には設定値にしてもよいです。

## 全身絵からの切り出し

### 1. 頭から上半身

画像の一番上、つまり頭の上から下方向へ `1024` を取る。

出力:

```text
1024x1024
```

目的:

- 顔、髪、上半身を学習させる。
- 頭が切れないようにする。

### 2. 首から下

首から下を中心に `1024x1024` を作る。

目的:

- `head out of frame` のような構図を作る。
- 顔をあえて入れない胴体、服、体型の学習に使う。

補足:

- 首の位置を画像認識だけで推定できるならそれでよい。
- 難しい場合は、先にEVA02などで `head out of frame` を作れるような補助処理が必要かもしれない。
- ただしEVA02タグだけに頼らず、画像位置から推定できるならそれでよい。

### 3. 足元側

画像の一番下、つまり足先側から上方向へ `1024` を取る。

出力:

```text
1024x1024
```

目的:

- 下半身、足、靴、裾などを学習させる。
- 足が切れないようにする。

### 4. 全身絵のまま残す画像

全身絵全体も1枚残す。

目的:

- キャラクター全体の比率、服装、全身の雰囲気を学習させる。

候補サイズ:

```text
1536x640
```

または、横を削りすぎる場合:

```text
1344x768
```

考え方:

- 全身画像はできるだけ全体を残す。
- まずは `1536x640` 系が適している可能性が高い。
- ただし、横を削りすぎる場合は `1344x768` 系など、より全身が残る比率へ逃がす。
- 実際には設定されている規定サイズ候補の中から選ぶ。

## 横幅が1024より小さい場合

全身絵を高さ2200へ合わせた後、横幅が `1024` より小さい場合があります。

その場合:

- 不足分をどう扱うか検討する。
- 通常画像の「削りすぎる場合は余裕のある比率へ逃がす」考え方を使う。
- 横方向に余裕が必要なら、横幅が広がる候補サイズを選ぶ。

重要:

- 変に人物を切らない。
- 頭、足、体全体が残ることを優先する。

## 横幅が1024を超える場合

全身絵を高さ2200へ合わせた後、横幅が `1024` を超える場合があります。

この場合:

1. 横幅を必要に応じて削り、`1024` に合わせる。
2. その後、上から `1024x1024`、首から下の `1024x1024`、下から `1024x1024` を作る。
3. 全身絵として残す画像は、規定サイズ候補の中から全身が一番残るものを選ぶ。

## 全身絵判定について

全身絵かどうかの判定が必要です。

候補:

- EVA02タグを使う。
- 画像内の人物の縦長さ、足先まで写っているか、余白の取り方から推定する。
- 将来的には手動で「全身絵として処理」チェックボックスを用意してもよい。

最初は自動判定にこだわりすぎず、手動指定を併用してよいです。

## 出力命名案

全身絵から4枚作る場合、元ファイル名に区別名を付けると分かりやすいです。
並び順が名前順で【上半身　首から下　下半身　全身】と順番になるように、
元の画像名が「A」とすると、

例:

```text
A_1.png
A_2.png
A_3.png
A_4.png
```
とする。

## 実装時の注意

- 画面サムネと実際の出力画像は必ず一致させる。
- 元画像、サムネ、出力PNGを比較して確認する。
- 切り取り結果が悪い場合、警告だけで終わらせず、できるだけ良い処理へ自動で逃がす。
- 全身絵は1枚に押し込むのではなく、4枚生成する方針を優先する。
- `1024` や `2200` は将来的に設定値へ移せるようにする。
- 実装は一気にやらず、通常画像処理と全身画像処理を分けて進める。

## まず実装してほしい順番

1. 手動で「通常画像」「全身絵」を選べるUIを作る。
2. 通常画像の比率選択と最小クロップを安定させる。
3. 全身絵を高さ2200へ合わせる処理を作る。
4. 全身絵から `upper`, `body`, `feet`, `full` の4枚を生成する。
5. 生成画像を画面サムネで確認できるようにする。
6. manifestに派生元と派生種別を記録する。
7. 自動判定は最後に追加する。
