# 絵コンテOCRの全体像と判断の仕組み（ep7）

> このドキュメントは「コンテOCRがどういうアプローチで読み、読んだものをどう判断しているか」を
> 一度で把握するための索引。**次セッションが同じ議論を再発明しないため**に置く。
> 実装は `src/genzu_fix/conte.py`。手順は `docs/local-cli-runbook.md`。

## 0. 最重要の原則（ここだけは外さない）

問題は2種類あり、**混同すると無限ループになる**。必ず分けて考える。

| 種類 | 例 | 性質 | 対処 |
|---|---|---|---|
| **構造（幾何・ロジック）** | actionの文字がdialogueに入る／ページまたぎの番号ズレ／枝番の統合崩れ | **解ける。再発しない。有限** | コードで根治 |
| **崩し字の読み違い** | 滲んだ1文字が芦か芹か／コマ数が読めず `3+?` | 確率的・毎回ブレる | review2で人が確定 |

> **フィールド（action/dialogue/time）は「文字が用紙のどの列に書かれているか」で機械的に決まる。**
> モデルに「どの欄か」を選ばせない。違う欄に入るのは読み取り能力でなく**列の切り方（幾何）の問題**。
> ＝「actionの内容がdialogueに入る」は構造バグであって、OCRの限界ではない。

## 1. パイプライン（3段＋訂正レイヤー）

```
PDF ─render→ ページPNG ─extract2→ コマ別JSON ─consolidate→ カット別CSV ─review2→ 訂正済みCSV(正)
              work/         conte_frames     conte_v2_         conte_v2_ep7.
              conte_pages   _v2_ep7.json      ep7.csv           corrected.csv
                              ↑Claude Vision   ↑決定論ロジック    ↑人が手で確定
```

| 段 | コマンド | 何をする | 誰が決める |
|---|---|---|---|
| render | `conte render --pdf <pdf>` | PDF→ページ画像化 | 機械（確定的） |
| extract2 | `conte extract2 --pages-dir work\conte_pages` | 列ごとに切り Claude Vision で文字起こし | **モデル（確率的）** |
| consolidate | `conte consolidate` | コマ→カットに統合・枝番判定 | 機械（確定的） |
| verify | `conte verify` | 構造不変条件を機械チェック（API不要） | 機械（確定的） |
| review2 | `conte review2 --pages-dir work\conte_pages` | エディタで人が通し読み・修正・書出し | **人（確定）** |

## 2. OCR（extract2）の中身

1. **画像処理で表を切る**（numpy、APIなし）。`detect_grid` が濃い横線＝行（コマ）、縦罫線＝列境界を検出。
2. **列ごとに別画像で渡す**。各コマを `左(番号+絵) / action欄 / dialogue欄 / time欄` の**4枚**に分割。
   - フィールドは「どの列の画像か」で決まる。モデルは各画像の中身を書き起こすだけで、欄をまたいで動かせない。
3. **用語集を文脈として注入**。`runs/conte_glossary_ep7.md`（ep7の人名・カメラ用語・尺記法・考証）を
   プロンプトに差し込む。崩し字は「何が書かれうるか」を先に知っていると誤読が激減する。
4. **出力**：カットごとに `cut_label / action / dialogue / se / time / numbered / characters / confidence / notes`。

### 列境界はep7用紙で固定（EP7_COLS）
- 本編ページのオーバーレイで目視確認済み：`pic|act=0.50, act|dia=0.70, dia|time=0.90`。
- `conte.py` の `EP7_COLS = "0.50,0.70,0.90"` を `--cols` の既定にしてある＝**毎回同じ位置で切れ、検出ノイズで動かない**。
- 用紙が違うとき／確認したいとき：
  - `--cols auto` で自動検出に戻す。
  - `--debug-crops <dir>` で API を叩かず、各ページに **列境界線を重ねた `_columns_overlay.png`**（赤=pic|act / 緑=act|dia / 青=dia|time）を保存。線が用紙の縦罫線に乗っていれば切り方は正しい。乗っていなければ読み取った比率を `--cols 0.xx,0.yy,0.zz` で固定する。
- 前付け（表紙・扉）は表が無いので `cuts:[]` を返す（正常）。最初のカットは page_003 から。

## 3. 読んだものをどう「判断」しているか

### A. 機械が決める（決定論・APIに依存しない）
- `numbered` フラグ → `_attach_continuations`：丸番号が描かれていない続きコマ（scene欄が縦棒｜だけ）は
  **直前カットへ統合**。これで**ページまたぎの番号ズレが根治**（モデルの番号推測に依存しない）。
- `consolidate`：`runs/cut_board_map_ep7.csv` の正規枝番リストだけ残し、それ以外のA/Bは基数へ統合。
  グループキーは `(page, 番号)` ＝**別ページの同番号は混ざらない**。
- confidence：結合カットは最も低い値に合わせる（保守的）。

### B. モデルが決める（確率的・毎回ブレる）
- 崩し字の文字起こし、人名同定、自信度（high/medium/low）の自己申告。
- review2の色：赤=low/文字化け、黄=medium、緑=high。**「人が見て」のフラグであって正しさの保証ではない**。

### 正データはどこか
- raw（機械読み＝`conte_v2_ep7.csv`）は不変。人の訂正は `conte_v2_ep7.corrected.csv` が**唯一の正**。
- OCRは下書き、人のレビューが確定、という分担。以降の原図理解・プロンプト設計は corrected を読む。

## 4. verify の読み方（再OCR後に必ず通す）

`conte verify` は構造不変条件を機械チェックする（API不要・一瞬）。出る項目と意味：

| 項目 | 意味 | 対処 |
|---|---|---|
| `dialogue混入` | dialogue欄に OK/A.P./承認印/兼用/PAN 等＝セリフでない | **0件であるべき**。出たら列幾何を `_columns_overlay.png` で点検 |
| `time混入` | time欄が秒+コマでない | `3+?` `f` 等は **time列内の不確実読み**（action漏れではない）→ review2で確定。`1+1/2` は「+12」等の崩し字誤読の疑い |
| `枝番` | 想定外の枝番／正規枝番の欠落 | 欠落は該当ページで A/B を補う |
| `欠番` | 連番の抜け | 複数回の再OCRで**毎回**欠番なら台本が本当に飛ばしている可能性大。単発なら読み落とし |

## 5. ep7 の確定事実（解釈し直さない）

- **枝番は正規6本だけ**：`123 / 213 / 216 / 229 / 239 / 295`（genzuファイル名で裏取り済み）。
  続きコマに勝手に枝番を振らない。**統合したら資料が無価値になる**。
- **尺＝秒+コマ**（24fps）。`4+12`=4秒12コマ。`4 1/2` のような分数読みは誤り。
- **人名は ep7 登場の面々から**（`conte_glossary_ep7.md` §1）。短い手書きを長い正式名に拡張しない。
  例：「扉に錠をかける◯◯の手もと」は **花家爺**（芦花婆さんではない）。
- 前付け（表紙・扉）は page_001〜002、本編は page_003 から。

## 6. 履歴（この設計に至った経緯の要点）

- 当初は右側（action+dialogue+time）を**1枚にまとめて**渡し、モデルに欄を振り分けさせていた
  → OK/A.P.（承認印）がdialogueに、action中の `F.I.(1+12)` の (1+12) が time に化ける混入が頻発。
- 「どの列に書かれているか＝そのフィールド」という原則に立ち返り、**列ごとに別画像**へ変更。
  境界はep7用紙で目視固定（0.50/0.70/0.90）。
- 結果（2026-06 再OCR, 305カット）：**dialogue混入 0件**。残る異常20件は time の崩し字・枝番の読み落とし・
  欠番＝崩し字の尻尾のみ。＝「列の取り違え」という無限ループは構造的に決着し、残りは review2 の有限作業。
