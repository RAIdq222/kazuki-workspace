# 背景原図のAI線画化パイプライン — 社内共有用まとめ（全体像）

> 何をする仕組みか／全体の地図／各工程の中身と「どこが工夫で、どこが面白いか」。
> 上から順に話せる構成。各セクション＝スライド1枚を想定。

---

## 0. 一言で
商用アニメ「尚善」の**背景原図(PSD)**を、**AIで白黒線画のドラフトに整え、元のPSDに
「AI原図修正」レイヤーとして返す**仕組み。**入口も出口もPSD**なので、美術さんの普段の
作業にそのまま溶け込む。ep7 規模で **245カット / 217原図 / 8班**。

ポイントは「AIは“下処理”担当・最終判断は人」「現場のフォーマット(PSD)に合わせる」の2点。

---

## 1. 全体の地図（4ステージ）
この仕組みは「生成だけ」ではない。**“何を描くべきかを理解する”前半**と、**“ズレなく描いて返す”後半**の二段構え。

```
 ① 理解（何を描くべきか）        ② 設計（どう描かせるか）
   一次資料を読み解く               プロンプトを3層で組む
   ・香盤表 / 美術ボード             ・GLOBAL（作品共通）
   ・絵コンテ(手描き)→Vision        ・SCENE （場所の語彙・考証）
   ・設定資料 / シナリオ            ・CUT  （時間天気・消すもの）
          │                                │
          └──────────────┬─────────────────┘
                         ▼
 ③ 生成と復元（ズレを設計で消す）     ④ 人が仕上げる（作業台）
   PSD→背景だけ抽出→出力寸ぴったり    コンソールで
   入力→GPT Image 2→厳密に逆変換で     見比べ→調整→生成/リテイク→OK
   原図画角へ→PSDに1レイヤー追加        （横並び/スライダー/重ね透過）
```

前半（①②）が「頭脳」、後半（③④）が「機構と現場」。**面白さは両方にある**。

---

## 2. パイプライン全7工程（機構の背骨）
1. **PSD取り出し** … 背景作画レイヤーだけを自動選択してPNG化（`psd_export`）
2. **入力整形** … GPT Image 2 の**出力解像度ぴったり**の入力を作る（`image_aspect`）
3. **ヘッダー除去** … 管理ヘッダー帯だけ落とす（撮影用の余白は残す）（`frame`）
4. **生成** … Higgsfield 経由で GPT Image 2（`gpt_image_2` / 2k / high）
5. **復元** … 入力を作った処理の**厳密な逆**で原図画角へ戻す
6. **PSD差し込み** … 「AI原図修正」レイヤーを最前面に追加（リテイクは _02,_03…）
7. **台帳記録** … 何の原図・ボード・プロンプト・パラメータで生成したか1行追記（`ledger`）

---

## 3. 面白ポイント① 「理解」を作り込む — ここが実は本体
生成より前に、**「このカットは何で、背景に何が要って、何を消すべきか」を一次資料から詰める**。
ここを機械化の境界で割っている：

- **機械で取れる所は自動**：場所 / 時代 / 構成物 / 時間 / 天気 は、**香盤表・美術ボード名・
  シーンプロファイル**から機械抽出（`cut_scene_info` の A/B層）。
- **コンテ依存の所はAIで補う**：「この場面で何が起きているか(situation)」「消す人物・持ち物(remove)」は
  カット固有なので、**絵コンテから読む**しかない（C層）。

### 絵コンテ OCR（手描き・テキスト層なしを構造化）
ep7のコンテは**スキャンPDFで文字データが無い**。そこで：
PDF→ページPNG化→**Claude Vision で構造化抽出**→カット番号でCSVへマージ。
- 抽出は「人物＋持ち物＝消す対象 / 環境物＝残す背景」を**検品ルールと同じ基準**で出す。
- **用語集を注入**して読み精度を上げる（登場人物の呼称・絵コンテ用語・尺記法・場所の統一表記）。
- **ページ跨ぎの連番ずれ**（縦棒の継続・前付けページ）を決定的ロジックで補正。

### 場面理解：3観点を“あえて食い違わせて”裁定する（いちばん面白い）
「理解」を1人のAIに任せると結論が丸まる。そこで**3つの役を別コンテキスト（別エージェント）で走らせ、
見せる入力を物理的に遮断**して、わざと違う視点を出させる：

| 役 | 見せる入力 | 役割 |
|---|---|---|
| 観点1 原図に忠実 | **原図PNGだけ**（コンテ/ボード/設定は隠す） | 絵に在るものだけを言う |
| 観点2 演出 | **絵コンテ＋前後カットだけ**（原図は隠す） | 芝居・意図を言う |
| 観点3 様式・考証 | **原図＋ボード＋設定資料**（コンテは隠す） | 時代考証・様式を言う |

→ 最後に**司会役が3観点を突き合わせて裁定**し、「概要≤3行／詳細／要確認アラート」を出す。
**独立性は“入力の遮断”で担保する**のが肝（同じ文脈で3役を演じると必ず結論が揃ってしまう）。
さらに考証(grounding)では「道観＝南岩宮（武当山南岩）」のように**一次資料に紐づけて根拠を持たせる**。

## 4. 面白ポイント② プロンプトは“3層”で組む（汎用 → シーン → カット）
毎回ゼロから書かず、**3層を積む**。EN（モデル入力）と JP（人の確認用）を**対**で出す。

| 層 | 中身 | 出どころ |
|---|---|---|
| **GLOBAL（作品共通）** | 役割＝「修正パス」（構図保持＋パース/構造修正＋品位向上）、白黒線質、消す/残す基準、レジスト厳守 | 固定文 |
| **SCENE（場所固有）** | 時代・建築・構成物の語彙、避けるもの(和風/西洋/ファンタジー禁止) | `scene_profiles/*.json` |
| **CUT（カット固有）** | 時間/天気の**線処理**への翻訳、場面(situation)、消すもの(remove) | 香盤表＋ボード名＋コンテ |

考え方のキモ：
- **「描き直し」でなく「なぞって整える(trace-and-clean)」を最優先**（構図の再解釈を防ぐ）。
- **色は参照しない**。ボードは「時代・構造・パース」の手がかりとしてだけ見る。
- **白黒だから夜・雨は“色”でなく“線の処理”に翻訳**（夜＝陰影輪郭の密度、霧＝遠景を疎に）。
- 「人物＝消す／環境＝残す」を**検品(QC)の文言と一語一句そろえる**（生成と検品の一貫性）。

## 5. 面白ポイント③ ズレを“後で直す”でなく“原理的に起こさない”
**問題**：AIは入力をなぞらず**描き直す**ので、生成結果が原図と微妙にズレる（位置・スケール）。

**普通の発想**：出力を後から位置合わせ補正（後追いワープ／正規化）。

**効いた発想**：そもそも**入力画像を「AIの出力解像度ぴったり」に作る**。
- 入力と出力が**同じピクセルグリッド**なら、絵は出力でも同じ整数座標に居る。
- だから「入力を作った処理の**逆**」で戻せば**理論上ずれようがない**。
- 例えるなら、**元絵の方眼ぴったりに切ったトレペに描く**。紙を合わせておけば線は必ず元の位置に戻る。
- GPT Image 2 の実出力寸は公称比とズレる（3:2でも 2048×1360 等）ので、**実測した出力寸の表**を持って合わせる。

さらに「ズレ」を2要因に**分解**：
- 要因(i) **生成の描き直し由来**（原理的に残る）→ プロンプトで「構図厳守」と抑える。
- 要因(ii) **余白付けと切り戻しの規約不整合**（系統誤差）→ 規約統一で**消す**。

→ ラウンドトリップ実測で**ズレ dx=0, dy=0**。「補正でごまかす」から「**設計でズレを消す**」への転換が肝。

## 6. 面白ポイント④ PSDで取って、PSDで返す（現場に溶け込む）
- 美術は普段 Photoshop/PSD。だから**新ツールを覚えさせない**。AIは「レイヤーが1枚増えるだけ」。
- **レイヤー選択の賢さ**：原図PSDには背景の絵以外に**指示書き・パース補助線・タップ穴・セル参考・
  カメラ枠**が同居。これを自動で見分け、`BG→LO→背景`の順で**背景作画だけ**抜く
  （指示/セル参考/Camera等は名前で除外、別ブックBOOKも既定除外）。「絵だけ」をAIに渡すのがキモ。

---

## 7. 面白ポイント⑤ 「眺める道具」でなく「捌く作業台」（コンソール）
ブラウザ上で全カットを一気通貫：
- 原図と結果を**見比べる**：**横並び／スワイプ比較／重ね合わせ透過**の3モード（＋左右入替・拡大）。
- プロンプトをその場で**調整 → 生成 / リテイク**（進捗表示・履歴）。
- 原図を直したら**取り直し**（背景自動検出 / 見たまま を選択）。
- **OK / 要修正**を記録（状態は永続化）。**PSDを直接開く**ボタンも。
- 上部に**話数のあらすじ・シーン展開・主要な美術ボード**（登場カット数順）。
- 美術ボードは**PSD/TIFFでもPNG化して表示**、ファイル名の**表記ゆれ（全半角・空白・拡張子）も吸収**。

---

## 8. 面白ポイント⑥ 「記憶」でなく「ファイル」を事実にする（並行作業の土台）
AIセッションは**別コンテナ・別履歴で記憶を共有しない**。だから事実は会話でなく**git上のファイル**に固定：
- `CLAUDE.md`（全セッション自動で読む索引＋鉄則）／`docs/sessions.md`（誰が何を・決定・未決）。
- 再利用手順は**スキル化**（例：原図の読み方 `/read-genzu`）。渡すときは「考えて」でなく「**この手順をなぞって**」。
- PSDはバイナリで読めない → **コンソールと同じ関数**でPNG化し、どの環境でも同じ原図を確認できる。

## 9. 規模・現状・残課題
- **規模**：245カット／217原図／8班。シーンは森(夜127/朝34/よどんだ朝16)・復活の儀25・道観17・寝室15 ほか。
- **整備**：シーンプロファイル 238/245(97%)。レジストは実測ズレ0。コンソールは全カット運用可。
- **残課題**：絵コンテ→situation/remove の全カット充足、美術ボード2枚目入力の本採用、
  カット個別の場面記述、検品(QC)の自動フラグ（台帳に枠は用意済み）。

---

## 10. 持ち帰り（3行）
1. **生成の前に“理解”を作り込む**：機械で取れる所は自動、コンテ依存はVisionで。3観点を**あえて食い違わせて裁定**。
2. **補正でなく設計でズレを消す**：入力をAI出力の解像度に合わせ、逆処理で厳密に戻す（dx=dy=0）。
3. **現場に溶け込ませる**：入口も出口もPSD。プロンプトは層で再利用（色でなく時代と構造）。人が最後に判定。
